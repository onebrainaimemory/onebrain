---------------------------------------

---------------------------------------
INPUT TYPES
---------------------------------------

The merge engine processes:

1. existing memory_items (active)
2. candidate memory_items (status = "candidate")

Each memory item has:
- type (fact, preference, decision, etc.)
- title
- body
- source_type
- confidence

---------------------------------------
OUTPUT
---------------------------------------

The engine must:

1. update canonical memory
2. mark duplicates
3. detect conflicts
4. assign confidence
5. create brain version snapshot

---------------------------------------
STEP-BY-STEP MERGE LOGIC
---------------------------------------

STEP 1 – NORMALIZATION
- trim text
- lowercase for comparison (do not modify stored original)
- remove duplicate whitespace

STEP 2 – DUPLICATE DETECTION

Two items are duplicates if:
- same type
- AND similar title (string similarity OR exact match)
- AND similar body

If duplicate:
- keep higher confidence
- merge sources
- mark lower one as "archived"

STEP 3 – CONFLICT DETECTION

A conflict exists if:
- same type
- same semantic topic
- but contradictory body

Example:
- "User prefers long answers"
- "User prefers short answers"

Action:
- mark both as "conflicted"
- do NOT auto-resolve
- keep both

STEP 4 – CONFIDENCE HANDLING

Assign base confidence:
- user input: 1.0
- system inference: 0.7
- external AI extraction: 0.6

If multiple sources agree:
- increase confidence (max 1.0)

If conflict:
- decrease confidence

STEP 5 – CATEGORY PRIORITY

Priority order:
1. user confirmed
2. user direct input
3. repeated AI extraction
4. single AI extraction

STEP 6 – UPDATE CANONICAL STATE

Update:
- brain profile (only if high confidence)
- memory_items table
- entity links if relevant

STEP 7 – SNAPSHOT

After successful merge:
- create new brain_versions entry
- store canonical snapshot JSON

---------------------------------------
EXPLAINABILITY REQUIREMENT
---------------------------------------

Each merge action must produce a log entry:
- action type (merge, conflict, archive)
- involved memory IDs
- reason
